# 文字格斗游戏

## 功能一：登录注册

描述：游戏拥有登陆、注册的功能，登录成功之后启动游戏~

## 1. 用户类：

属性：id、用户名、密码、状态

**各属性描述：**

1. id：

* 用户无法设置，是自动生成的，格式为：heima+5位数字的随机数

2. 用户名username：

* 用户名唯一
* 长度必须在3 ~ 16位
* 只能由字母、数字组成，不能是纯数字

3，密码要求：

* 长度3 ~ 8位
* 只能是字母加数字的组合，不能有其他字母

4，状态state

* 默认是可使用状态
* 如果密码连续输错3次，修改为不可使用状态

## 2. 主界面：

```java
System.out.println("╔════════════════════════════════╗");
System.out.println("    🎮 欢迎来到文字格斗游戏 🎮   "); 
System.out.println("╚════════════════════════════════╝");
System.out.println("请选择操作：1登录 2注册 3退出");
```

#### 2.1 注册功能：

* 注册时候，各属性需要满足要求
* 密码键盘输入两次，两次一致才可以进行注册

#### 2.2 登录功能：

1. 键盘录入用户名
2. 键盘录入密码
3. 键盘录入验证码
4. 登录最多重试三次，三次错误账号锁定

**验证要求：**

​	用户名如果未注册提示：用户名未注册，请先注册

​	用户被锁定提示：用户xxx已经锁定，请联系黑马程序员官方客服：XXX-XXXXX

​	验证码错误提示：验证码输入错误，请重新输入，并生成一个新的验证码

​	判断用户名和密码是否正确，有3次机会，满3次账户锁定。

#### 2.3 验证码规则：

​	长度为5

​	由4位大写或者小写字母和1位数字组成，同一个字母可重复

​	数字可以出现在任意位置

比如：aQa1K

## 功能二：文字格斗游戏

## 1. 游戏概述

### 1.1 游戏名称

文字格斗游戏

### 1.2 游戏类型

基于控制台的回合制文字RPG游戏

回合制：

​	你打我一下，我打你一下

### 1.3 游戏目标

玩家通过创建角色、分配属性、使用技能战胜一系列敌人，获得胜利场次并提升角色属性。

### 1.4 核心机制

- 角色创建与属性分配系统

  - 角色创建：创建人物的对象（我方/多个敌人）

  - 属性分配：血量，攻击，防御等属性，我们需要给这些属性进行加点

    20个属性点：----> (血量，攻击，防御)

- 回合制战斗系统（战斗过程）
  - 我打敌人一下
  - 判断敌人的生命值是否为0
  - 敌人打我一下
  - 判断我方的生命值是否为0
  - 如果为0，游戏结束
  - 如果不为0，重复上面打斗的过程
- 技能系统
  
  - 普通攻击 / 技能攻击
- 胜场积累与成长系统
  - 胜场积累：不是打一个敌人就结束了，在游戏当中，我需要打多个敌人，还需要把胜利的次数记录下来
  - 成长系统：当我胜利的时候，得到一些奖励（道具，属性点）

## 2. 角色系统

### 2.1 角色创建

* 角色名称由登录而来（name）

- 角色的属性由初始属性和分配属性得来，分配20点初始属性到三个属性：
  - 生命值（每点+10HP）
  - 攻击力（每点+2ATK）
  - 防御力（每点+1DEF）
  
  人物属性的计算公式：
  
  ​	初始属性 + 分配点数 * 比例

### 2.2 角色属性

| 属性               | 描述                | 基础值                    | 成长值   |
| ------------------ | ------------------- | ------------------------- | -------- |
| 角色名             | 玩家的角色登录而来  | 无                        | 无       |
| 生命值(HP) / maxHP | 角色生存能力        | 初始血量100 + 分配点数×10 | 每3胜+30 |
| 攻击力(ATK)        | 影响伤害输出        | 初始攻击10 + 分配点数×2   | 每3胜+5  |
| 防御力(DEF)        | 减少受到的伤害      | 初始防御0 + 分配点数×1    | 每3胜+3  |
| 技能列表（集合）   | 玩家技能列表参见2.3 | 无                        | 无       |

### 2.3 玩家技能列表

| 技能名称 | 消耗 | 效果                 | 描述               |
| -------- | ---- | -------------------- | ------------------ |
| 普通攻击 | 无   | 造成基础伤害         | 标准攻击方式       |
| 强力一击 | 10HP | 造成180%攻击力的伤害 | 高伤害但消耗生命   |
| 生命汲取 | 10HP | 恢复0-20点生命值     | 风险回报型恢复技能 |

### 2.4 敌人类型

| 敌人名称 | 生命值 | 攻击力 | 防御力 | 技能（变量）                                           |
| -------- | ------ | ------ | ------ | ------------------------------------------------------ |
| 初级战士 | 80     | 15     | 10     | 猛击（150%伤害）                                       |
| 敏捷刺客 | 60     | 20     | 5      | 快速攻击（2次50%伤害）                                 |
| 重装坦克 | 120    | 10     | 20     | 防御姿态（下回合伤害减半） buff（ boolean defendding） |
| 神秘法师 | 70     | 25     | 8      | 火球术（180%伤害）                                     |

## 3. 成长系统

### 3.1 玩家成长系统

每胜利三场，玩家属性增加`HP+30, ATK+5, DEF+3`

### 3.2 怪物成长系统

第一场战斗：怪物是基础属性

玩家每连胜一场：怪物属性增加HP+10, ATK+3, DEF+2

## 3. 战斗系统

### 3.1 伤害计算公式

- 基础伤害公式：`伤害 = 攻击力 - 防御力 `
- 最小伤害：1点  
- 技能伤害：`伤害 = 攻击力 * n% - 防御力`

### 3.2 单场战斗流程

1. 战斗开始，显示双方状态（生命值）

2. 玩家回合：选择行动（ 1普通攻击/ 2强力一击/ 3生命汲取 ）

3. 判断敌人是否被击败（判断敌方血量是否为0）

4. 敌人回合：选择行动（ 50%的几率普通攻击 / 50%的几率技能攻击 ）

5. 判断玩家是否被击败（判断玩家血量是否为0）

6. 战斗结算

   玩家胜利：恢复生命值继续下一场战斗

   玩家失败：游戏停止

### 3.3 战斗结算

- 胜利时：
  - 恢复20-40点生命值
  - 胜场数+1
  - 每3胜获得属性提升
- 失败时：游戏结束

## 4. 游戏界面与交互

### 4.1 用户输入

| 输入场景 | 有效输入                | 验证规则        |
| -------- | ----------------------- | --------------- |
| 角色名称 | 登录时候的名称          | 不验证          |
| 属性分配 | 整数                    | 0≤输入≤剩余点数 |
| 战斗选择 | 1, 2, 3（选择释放技能） | 仅接受指定数字  |
| 继续游戏 | y/n                     | 不区分大小写    |

## 5. 游戏流程

### 5.1 主流程

1. 显示游戏标题
2. 创建玩家角色（名称+属性分配）
3. 创建成功，展示玩家的所有属性和技能列表
4. 创建多个敌人列表
5. 准备战斗（依次跟多个敌人战斗）：
   - 5.1 重置敌人的属性，敌人属性每场HP+10, ATK+3, DEF+2（敌人：越来越打）（第二场的时候）
   - 5.2 随机选择敌人(Random)
   - --------战斗开始（重复）---------
   - 5.3 战斗开始，显示双方状态（生命值）
   - 5.4 玩家回合：选择行动（ 1普通攻击/ 2强力一击/ 3生命汲取 ）
   - 5.5 判断敌人是否被击败（判断敌方血量是否为0）
   - 5.6 敌人回合：选择行动（ 50%的几率普通攻击 / 50%的几率技能攻击 / 不同的敌人采取不同的技能进行攻击）
   - 5.7 判断玩家是否被击败（判断玩家血量是否为0）
   -    round++
   - --------战斗结束（重复）--------
   - --------下面开始战斗结算--------
   - 5.8 跟一个敌人的战斗结束后，玩家胜利（恢复生命值继续战斗）玩家失败（游戏停止）
   - 5.9 每3胜提升属性
   - 5.10 询问是否继续
10. 整个游戏结束，最终结算

## 功能三

## 1. 忘记密码

**主要业务：**

根据手机号码重置密码

**代码提示：**

用户类：新增属性手机号

手机号规则：纯数字，长度11位，只能以数字1为开头

### 1.1 具体执行规则

```java
System.out.println("╔═════════════════════════════╗");
System.out.println("    🎮 欢迎来到文字格斗游戏 🎮   ");
System.out.println("╚═════════════════════════════╝");
System.out.println("请选择操作：1登录 2注册 3忘记密码");
```

* 注册的时候需要输入手机号

* 登录的时候无需输入手机号
* 忘记密码的规则如下：
  * 选择3，进入忘记密码操作
  * 输入用户名，查询用户名是否存在
  * 不存在：提示当前用户名未注册
  * 存在：输入手机号，并验证
  * 手机号正确，输入新密码
  * 修改密码

## 2. 蓝条

增加蓝条属性 mp(当前蓝量) maxMP（最大蓝量）

规则：

​	释放技能的时候扣蓝，每次10点蓝量，如果蓝量为0，无法释放技能

​	打败一个敌人，恢复最大蓝量的30% 

## 3. 嘲讽

如果战斗失败，增加敌人嘲讽话术

规则:

​	可以提前把敌人嘲讽的话语，拿一个集合或者数组存储

​	当敌人将我方打败之后，需要从容器中随机抽取嘲讽的话术并进行打印。（如果兄弟你的文明人，不会嘲讽话术可以借助AI）
